urlFormatproperty 
urlFormat:String  [read-write]

A string that determines the URL generated by this rule.

For rules that have a notion of application state embedded in the URL, the format string must contain an asterisk " which will be substituted with the that state when the URL is generated.

If urlPattern is not specified, it is defaulted from the urlFormat property, and a default value-group regexp is substituted for the " character. This default regexp group will match all non-delimiting URL characters.

This property can be used as the source for data binding.

urlPatternproperty 
urlPattern:String  [read-write]

A regular expression that will restrict this rule to only be active for a URL that matches it.

In the special case of rules that have a notion of an application state to be extracted from the URL, the regular expression must contain a single parenthesized group, whose matched contents yield this state.

If not given or set to null this property is effectively defaulted by the value of the urlFormat property.

This property can be used as the source for data binding.

checkReadiness()method 
protected function checkReadiness(e:Event):void

Check this rule to determine whether it's ready, setting the ready property to reflect this. This function must be called at any time that the readiness status of the rule may have changed.

This method should be overridden by subclasses as needed, to impose additional conditions on when the rule has been sufficiently initialized to run. Overrides must set ready to false if their conditions are not met, otherwise call super.checkReadiness(). Parameters

matchUrlPrefix()method 
public function matchUrlPrefix(url:String):String

Determine whether this rule is able to consume and interpret all or some prefix of a given URL as a valid application state. If it can do so, return the initial portion of the URL which can be consumed by this rule. If the rule is "not interested" in any prefix of the URL, return the empty string. Otherwise, the URL begins with a string that is known to be invalid, in which case return null.

Parameters
url:String — the URL whose prefix is being matched.

validateState()method 
protected function validateState():void

Validate this rule such that its URL and title correspond to the application state, as determined by whatever URL/state mapping this rule defines. Calls computeApplicationState() to do the rule-specific work.

If the rule is unable to validate its state, it throws a StateNotAvailableError instance to indicate this failure.
